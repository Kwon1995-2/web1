[Hyperledger] 저장소 제공 서비스

  • 적용기술 : Hyperledger, 암호화, 해시, 인증 등등

  • 적합업무 : 클라우드 제공 서비스, 개인데이터 관리

  • 적용곤란업무 : 개인과 개인 간의 매핑(즉, 퍼블릭 블록체인과 호환성)
    • ↳ 개인이 서버를 항시 제공할 수 있을 가능성 적음

  • 위험/장애 : 기존 중앙관리시스템과 마찬가지로 중앙서버 해킹에 대비해야 함

  • 개발/운영 수월성 : 중간
    • ↳ 개인이 여러 선택지 중 이용할 클라우드 서비스를 선택
    ↳ 해당 클라우드 서비스 이름, 기간 등만 블록에 저장

  • 개발 비용 : 블록체인 구현은 오픈소스 활용, 고객에게 보여질 GUI의 개발이 다소 어려움
    1. 서비스 제공자를 선정
    2. 사용자가 응용 프로그램에서 서비스 제공자, 기간, 사용공간 등을 명시하면 응용 프로그램은 피어그룹에 트랜잭션 요청
    3. 피어그룹이 검증한 후, 정렬자에게 허가 요청
    4. 정렬자로부터 허가 요청을 받으면 피어그룹은 트랜잭션 실행 후, 저장
    5. 사용자는 저장공간을 명시된 계약에 따라 사용 가능

  • 효과
    • 데이터 분산 저장 및 관리
    • 네이버, 구글 같은 IT기업이 아니어도 참여 가능
    • 엣지 컴퓨팅 활용 가능성
